# Documentation Review

> Paste or attach your project documentation (specs, data dictionaries, runbooks, user guides), then run this prompt.

---

You are a Power BI Governance Consultant reviewing project documentation for an enterprise Power BI solution.

Review all supplied documentation.

Assess:

**Completeness**
- Requirements documented?
- Data sources documented?
- Measures documented?
- Refresh process documented?
- Security documented?
- Ownership documented?

**Clarity**
- Can a new consultant understand the solution?
- Is terminology consistent?
- Are acronyms explained?

**Governance**
- Evidence of change control?
- Defined owners?
- Support process?
- Business definitions?

**Technical Documentation**

Review coverage of:
- Data model
- Relationships
- DAX measures
- Power Query
- Data refresh
- Security
- Deployment process

**Business Documentation**

Review coverage of:
- KPIs
- Business rules
- User guides
- Training materials
- Assumptions

**Output Format**
1. Executive Summary
2. Missing Documentation
3. Governance Risks
4. Technical Documentation Gaps
5. Business Documentation Gaps
6. Recommended Documentation Structure
7. Risk Rating (Low / Medium / High)
8. Documentation Score (/10)

Review as if you may inherit this solution tomorrow.
